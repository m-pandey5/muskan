About me

I am currently in my final year of my Bachelors at I.I.T Delhi. I am the President of the Blockchain Society at I.I.T Delhi.

I have worked as a Cryptography Research Engineer at Encifher, and as a Backend Engineer at Umbra Privacy.

Projects

  • Ooty VM (BrainfuckVM) — ETHIndia 2024 — Developed Ooty VM, a complete STARK-based zero-knowledge virtual machine in Rust that proves computational integrity of arbitrary Brainfuck ISA programs. Implemented prover and verifier from scratch, including a custom FRI (Fast Reed-Solomon IOP) pipeline, univariate polynomial arithmetic, Merkle tree commitments, Fiat–Shamir channels, and all supporting finite-field cryptography, with no external cryptographic dependencies. Enhanced the reference BrainSTARK design by using univariate quotient polynomials and parallel Lagrange interpolation for improved proving efficiency.
  • Zk-Market — ETHGlobal Singapore 2024, Rootstock track winner — Built Zk-Market, a privacy-preserving marketplace that bridges Web2 and Web3 using proxy-based ZK-TLS and the Reclaim Protocol. Designed and implemented an end-to-end system that generates zero-knowledge proofs of Web2 credentials and user data (e.g., social activity, GitHub contributions) without exposing raw information, enabling trustless on-chain verification and automated incentive distribution via smart contracts. Integrated Reclaim libraries for off-chain attestation and Hardhat-deployed contracts for proof validation and marketplace logic.
  • proofPositive (Zcomp) — ETH Mumbai 2024, main-track winner in Zero Knowledge — Designed and implemented proofPositive (Zcomp), a modular compiler pipeline (inspired by CirC) that translates high-level programs written in C, Rust, Python, Solidity, and ZoKrates into optimized R1CS constraints for Groth16 proving. Developed language-specific frontends with AST parsing and CIRC-IR (EQC) intermediate representation, along with backend optimization passes, R1CS lowering, and proof synthesis using the bellman crate (supporting Groth16 and Spartan).
  • Quark Layer — BlocSoc R&D — A decentralised proof verification and aggregation layer. Faster proof verification guarantees using cryptoeconomic security of restaked validators (soft guarantee). A hard but slow proof verification guarantee using cryptographic security of zkVMs. Aggregation for multi-proof verification and reduced on-chain verification cost, cutting cost and latency by orders of magnitude for proof verification
  • stark101-rs — Rust implementation of Stark101 by StarkWare
  • Rust Blockchain — Educational Peer-to-Peer Blockchain — Built a functional peer-to-peer blockchain from scratch in Rust to understand core distributed ledger concepts. Implemented Proof-of-Work mining (SHA-256 with a difficulty target), block and chain sync over libp2p (TCP transport with Noise, Floodsub pub/sub, mDNS peer discovery), per-block validation, longest-valid-chain fork resolution, and multi-node synchronization. Designed a command-line interface supporting peer discovery (ls p), chain inspection (ls c), and block creation with network-wide propagation (create b).
